## Known Risks (Waived Review Findings)

- **W1 (cycle 3 F4)**: Wrapper writes to `/workspace/agent/.mnemon-metrics.jsonl` (per-agent-group, multi-writer if concurrent sessions). **Mitigation:** POSIX `O_APPEND` is atomic for writes ≤ PIPE_BUF (4096B); design's metric rows are well under. Builder validates atomicity by inspecting line counts during integration smoke. If interleaving observed, switch wrapper to per-session path via `MNEMON_SESSION_ID` env (single-line edit).
- **W2 (cycle 3 F6)**: Wrapper script `set -euo pipefail` may hard-exit on transient mkdir/flock failures. **Mitigation:** Failure mode falls through to hook's `failure-class.ts` SF6 discipline. Builder adds explicit error handling around `mkdir`/`flock` only if non-zero exits show up in metrics during shadow week.
- **W3 (cycle 3 Codex#5)**: Phase 2 graduation gate uses absence-of-unhealthy rather than positive heartbeat freshness. **Mitigation:** Operator weekly visual review (Phase 1 gate #4) catches systemic missing-metrics issues. Defer positive freshness gate to post-Phase-1 observability iteration.

### Task A1: Dockerfile mnemon binary install + wrapper symlink

**File:** `container/Dockerfile`
**Operation:** MODIFY
**Test file:** none — reason: Dockerfile is build-time artifact. Validated by image-build smoke test in D1 acceptance and `mnemon --version` smoke check.

**Approach:** Add ARG-pinned mnemon binary install with per-arch SHA256 verification (mirrors RENDER pattern at lines 118-127, adds checksum verify). After install, COPY the wrapper script (created in A2) to `/usr/local/bin/mnemon-wrapper.sh`, rename real binary to `/usr/local/bin/mnemon-real`, and symlink wrapper to the canonical PATH location `/usr/local/bin/mnemon`. Final `mnemon --version` smoke validates wrapper passes through to real binary.

Insert ARGs and RUN block AFTER the RENDER/RAILWAY/SUPABASE block (after Dockerfile line ~127). The COPY for the wrapper script happens after the install RUN block so the wrapper exists before the rename+symlink step.

**Code pattern:**

```dockerfile
# Pinned mnemon binary — auto-tracked by /update-container audit.
# Per-arch SHA256 from https://github.com/mnemon-dev/mnemon/releases/download/v0.1.2/checksums.txt
ARG MNEMON_VERSION=0.1.2
ARG MNEMON_SHA256_arm64=e9acb178843605a813520dacce10f8f3b3608df65841c238f0b3b768b74cd025
ARG MNEMON_SHA256_amd64=a18aa4304e4d904cb80e46a099efbd95bf0b1486381757be89cd234844603347

RUN ARCH=$(dpkg --print-architecture) \
    && SHA=$(if [ "$ARCH" = "arm64" ]; then echo "$MNEMON_SHA256_arm64"; else echo "$MNEMON_SHA256_amd64"; fi) \
    && curl -fsSL "https://github.com/mnemon-dev/mnemon/releases/download/v${MNEMON_VERSION}/mnemon_${MNEMON_VERSION}_linux_${ARCH}.tar.gz" -o /tmp/mnemon.tar.gz \
    && echo "$SHA  /tmp/mnemon.tar.gz" | sha256sum -c - \
    && tar -xzf /tmp/mnemon.tar.gz -C /tmp mnemon \
    && mv /tmp/mnemon /usr/local/bin/mnemon-real \
    && chmod +x /usr/local/bin/mnemon-real \
    && rm -rf /tmp/mnemon*

# Wrapper script (created in Task A2). Symlinked to canonical PATH location.
COPY mnemon-wrapper.sh /usr/local/bin/mnemon-wrapper.sh
RUN chmod +x /usr/local/bin/mnemon-wrapper.sh \
    && ln -s /usr/local/bin/mnemon-wrapper.sh /usr/local/bin/mnemon \
    && /usr/local/bin/mnemon --version
```

**ASSERT (build-time):**
- `sha256sum -c` PASSES for the downloaded tarball — checksum match required by [HARD] C10
- `mnemon --version` PASSES at build time, confirming wrapper transparently passes through

### Task A2: Container mnemon wrapper script

**File:** `container/mnemon-wrapper.sh`
**Operation:** CREATE
**Test file:** none — reason: bash wrapper integration-tested via Group B's hooks tests + Group D's smoke tests (`which mnemon`, recall block during shadow, write lock contention).

**Approach:** Bash wrapper script that centralizes phase gating (recall block during shadow), shell `flock(1)` on host-shared lock for write commands, and per-command observability events. Script logic verbatim from design's "Mnemon CLI wrapper" section. Reads phase from `/workspace/agent/.mnemon-rollout.json` (RO-mounted host file). Fails closed (defaults to `shadow` on missing/malformed JSON) per cycle 3 F3. Locks at host-shared path `${HOME}/.mnemon/data/${STORE}/.write.lock` per cycle 3 MF10.

**Code pattern:** See design.md "Mnemon CLI wrapper" section. Implementation is the bash code block verbatim, including:
- `set -euo pipefail` + the case statement
- Phase resolution with `jq -r` and `// "shadow"` fail-closed default
- Phase value validation: `case "$PHASE" in shadow|live|unhealthy) ;; *) PHASE="shadow" ;; esac`
- `recall|search|related` → shadow-block returns `{"results":[]}` empty JSON; live passes through
- `remember|link|forget|embed|gc` → `flock` write-lock then exec real binary
- `store|setup` → admin-locked + audited (cycle 3 Codex#4 fix)
- `status|viz|version|help|""|--version|--help` → unmediated passthrough
- Unknown subcommand → emit metric with `event_type: "unhealthy"` and `reason: "unknown-subcommand"` to `/workspace/agent/.mnemon-metrics.jsonl`, write stderr message, then `exit 2` (cycle 3 Codex#4 + drift check B9 — design.md Wrapper script: "unknown subcommands → exit 2, emit unhealthy event")
- `emit_metric()` writes JSONL to `/workspace/agent/.mnemon-metrics.jsonl`. Distinguishes `event_type: "turn"` (normal invocation) from `event_type: "unhealthy"` (unknown subcommand or wrapper internal failure).

**ASSERT:**
- Script returns `{"results":[]}` for `recall|search|related` when phase is `shadow`
- Script `exec`s flock + real binary for write commands (verified by `bash -x` trace showing flock is in argv before mnemon-real)
- Script writes one JSONL row per invocation to `/workspace/agent/.mnemon-metrics.jsonl`
- Script defaults to `shadow` (NOT `live`) when rollout JSON is missing/malformed
- Script exits 2 (not 0, not 1) when subcommand is unknown
- Unknown subcommand path emits `event_type: "unhealthy"` with `reason: "unknown-subcommand"` (NOT `event_type: "turn"` or `event_type: "error"`) — drift check B9

**Acceptance criteria:**
- [ ] File present at `container/mnemon-wrapper.sh`, executable bit set
- [ ] All decision branches in design's wrapper code block present
- [ ] `bash -n container/mnemon-wrapper.sh` (syntax check) passes
- [ ] Manual unit test: `MNEMON_STORE=test ./container/mnemon-wrapper.sh recall "x"` with no rollout JSON returns `{"results":[]}` (fail-closed)
- [ ] Manual unit test: `./container/mnemon-wrapper.sh unknownsubcmd` exits 2

**Approach:** Public API for the host module. `index.ts` exports `applyMnemonMounts` and `applyMnemonEnv` — both no-op when `containerConfig.mnemon?.enabled !== true`. When enabled: applyMnemonMounts pushes the `~/.mnemon` RW data dir mount and `data/mnemon-rollout.json` RO mount; applyMnemonEnv pushes `MNEMON_STORE` env (and optional `MNEMON_EMBED_*` when `embeddings: true`). `config.ts` re-exports the `MnemonConfig` type for downstream tests.

**Code pattern:**

```typescript
// src/modules/mnemon/index.ts
import path from 'path';
import { homedir } from 'os';
import type { ContainerConfig } from '../../container-config.js';
import type { Mount } from '../../container-runner.js'; // adjust import based on actual type location
import type { AgentGroup } from '../../types.js';

interface ApplyArgs<T> {
  agentGroup: AgentGroup;
  containerConfig: ContainerConfig;
  // T is whatever container-runner uses for the assembly (mounts array or args array)
}

export interface ApplyMountsArgs extends ApplyArgs<Mount[]> {
  mounts: Mount[];
}

export interface ApplyEnvArgs extends ApplyArgs<string[]> {
  args: string[];
}

const HOST_MNEMON_DIR = path.join(homedir(), '.mnemon');
const ROLLOUT_FILE = path.join(process.cwd(), 'data', 'mnemon-rollout.json');

export function applyMnemonMounts({ mounts, agentGroup, containerConfig }: ApplyMountsArgs): void {
  if (!containerConfig.mnemon?.enabled) return;

  // RW mount for the mnemon data dir — covers DB writes + write lockfile per-store.
  mounts.push({ hostPath: HOST_MNEMON_DIR, containerPath: '/home/node/.mnemon', readonly: false });

  // RO mount for the rollout state JSON — host owns it; container reads phase from here.
  mounts.push({ hostPath: ROLLOUT_FILE, containerPath: '/workspace/agent/.mnemon-rollout.json', readonly: true });
}

export function applyMnemonEnv({ args, agentGroup, containerConfig }: ApplyEnvArgs): void {
  if (!containerConfig.mnemon?.enabled) return;

  // MNEMON_STORE: per-group isolation (covers [HARD] C4).
  args.push('-e', `MNEMON_STORE=${agentGroup.id}`);

  if (containerConfig.mnemon.embeddings) {
    args.push('-e', 'MNEMON_EMBED_ENDPOINT=http://host.docker.internal:11434');
    args.push('-e', 'MNEMON_EMBED_MODEL=nomic-embed-text');
  }
}
```

**ASSERT:**
- `applyMnemonMounts` and `applyMnemonEnv` are no-ops (exit early, no side effects on `mounts` or `args`) when `containerConfig.mnemon` is `undefined` or `{ enabled: false }`
- When enabled: `mounts` gains exactly two entries (data dir RW + rollout JSON RO); no other groups affected
- When `embeddings: true`: both `MNEMON_EMBED_ENDPOINT` and `MNEMON_EMBED_MODEL` env vars pushed
- `MNEMON_STORE` always equals `agentGroup.id` (no other source of truth)

**Acceptance criteria:**
- [ ] `applyMnemonMounts` and `applyMnemonEnv` exported from `src/modules/mnemon/index.ts`
- [ ] No-op for `mnemon.enabled !== true` — verified by tests
- [ ] Mount paths exactly: host `~/.mnemon` → container `/home/node/.mnemon` (RW); host `data/mnemon-rollout.json` → container `/workspace/agent/.mnemon-rollout.json` (RO)

### Task A5: Host mnemon module — binary detection + ensureStore

**Files:**

- `src/modules/mnemon/store.ts` (CREATE)

**Operation:** CREATE
**Test file:** `src/modules/mnemon/index.test.ts` (covered by Task A6)

**Approach:** Host-side helpers. `binary.ts` exports `mnemonBinaryAvailable(): Promise<boolean>` that checks for `mnemon` in PATH (used by enable-mnemon.ts pre-flight). `store.ts` exports `ensureStore(storeName: string): Promise<void>` — idempotent wrapper that runs `mnemon store create <name>` and ignores already-exists errors. **Code pattern:**

```typescript
// src/modules/mnemon/binary.ts
import { execFile } from 'child_process';
import { promisify } from 'util';
const execFileP = promisify(execFile);

export async function mnemonBinaryAvailable(): Promise<boolean> {
  try {
    const { stdout } = await execFileP('mnemon', ['--version']);
    return /^mnemon version/.test(stdout.trim());
  } catch {
    return false;
  }
}

// src/modules/mnemon/store.ts
import { execFile } from 'child_process';
import { promisify } from 'util';
const execFileP = promisify(execFile);

export async function ensureStore(storeName: string): Promise<void> {
  try {
    await execFileP('mnemon', ['store', 'create', storeName]);
  } catch (err) {
    const msg = err instanceof Error ? err.message : String(err);
    // Idempotent: tolerate already-exists. Mnemon's exact error message for dupe should
    // be verified in test; for now, swallow any error containing "exists" or "already".
    if (/already|exists/i.test(msg)) return;
    throw err;
  }
}
```

**ASSERT:**
- `mnemonBinaryAvailable` returns `true` when binary is on PATH and `--version` succeeds; `false` on any error
- `ensureStore` is idempotent — calling twice with same storeName does NOT throw on second call
- `ensureStore` propagates non-already-exists errors (e.g., disk full, permission denied)

**Test cases:** See Task A6.

**Acceptance criteria:**
- [ ] `mnemonBinaryAvailable` and `ensureStore` exported
- [ ] `ensureStore` is idempotent (test asserts double-call doesn't throw)
- [ ] Both functions are async (return `Promise`)

**Pre-conditions:** None (no dependency on other A tasks; can develop in parallel with A4).

**Operation:** CREATE

**Approach:** Three small modules, all bun:test compatible:
- `binary-classes.ts` — exports constants `READ_ONLY_COMMANDS = ['recall','search','related','status','viz','version','help','--version','--help']` and `WRITE_COMMANDS = ['remember','link','forget','embed','gc']` and `ADMIN_COMMANDS = ['store','setup']`. - `rollout-reader.ts` — exports `readPhase(store: string): 'shadow' | 'live' | 'unhealthy'`. - `failure-class.ts` — exports `classifyError(err: Error): 'recoverable' | 'blocking'`. Heuristic-based: missing-binary, store-DB-inaccessible, schema-mismatch → blocking; Ollama-timeout, network-blip, single-write-failure → recoverable. Returns `{ class, reason: string }`.

**Approach:** Three hook-factory functions exported: `createMnemonPrimeHook(store)`, `createMnemonRemindHook(store)`, `createMnemonNudgeHook(store)`. Each returns an SDK-compatible hook function (signature varies by event). Per cycle 1 D16+D23, hooks DO NOT invoke mnemon CLI — they only return reminder text or emit metrics. **Schema-mismatch detection at SessionStart (covers SF11 — design.md):** Prime hook calls `detectSchemaMismatch(store)` once at session start and caches the result in a module-level `Map<string, SchemaCheckResult>` keyed by store. UserPromptSubmit and Stop hooks read from cache only — they never re-invoke detection on the hot path. Detection itself is a single `mnemon status` call (read-only, JSON-by-default — no `--json` flag); a non-zero exit or schema-mismatch error string in stdout/stderr triggers an unhealthy event one time per session. Subsequent hooks in the same session use the cached `ok | mismatch | unknown` verdict to short-circuit.

**Error logging (covers B8 — explicit WARN/ERROR per design Hook common discipline):** Every hook's catch branch must emit an explicit log line. Recoverable errors → `console.warn('[mnemon] <hook> recoverable: <reason>')`. Blocking errors → `console.error('[mnemon] <hook> BLOCKING: <reason>')` and emit unhealthy event. The hook still returns a safe default afterward; logging is for the host log-aggregator.

```typescript
// Pattern from: container/agent-runner/src/providers/claude.ts:947 (existing hooks)
// Adapted for: mnemon SessionStart/UserPromptSubmit/Stop hooks
import fs from 'fs';
import { execFile } from 'child_process';
import { promisify } from 'util';
import { readPhase } from './rollout-reader.js';
import { classifyError } from './failure-class.js';
import { emitTurnMetric, emitUnhealthyEvent } from './metrics-emit.js';

const execFileP = promisify(execFile);
const GUIDE_PATH = '/home/node/.mnemon/prompt/guide.md';

// Per-session schema-mismatch cache (cycle 2 SF11 — design.md).
// Keyed by store. Populated at SessionStart, read at every UserPromptSubmit/Stop.
type SchemaCheckResult = 'ok' | 'mismatch' | 'unknown';
const schemaCache: Map<string, SchemaCheckResult> = new Map();

async function detectSchemaMismatch(store: string): Promise<SchemaCheckResult> {
  try {
    // mnemon status emits JSON by default (no --json flag).
    const { stdout, stderr } = await execFileP('mnemon', ['status', '--store', store], { timeout: 3000 });
    if (/schema.*(mismatch|version|incompatible)/i.test(stdout + stderr)) return 'mismatch';
    return 'ok';
  } catch (err) {
    const cls = classifyError(err as Error);
    if (cls.reason === 'schema-mismatch') return 'mismatch';
    return 'unknown';
  }
}

export function createMnemonPrimeHook(store: string) {
  return async (input: unknown) => {
    const start = Date.now();
    try {
      const phase = readPhase(store);
      let guide = '';
      try { guide = fs.readFileSync(GUIDE_PATH, 'utf8'); } catch { /* missing guide is recoverable */ }

      // Schema-mismatch detection: run once per session, cache the result (SF11).
      if (!schemaCache.has(store)) {
        const verdict = await detectSchemaMismatch(store);
        schemaCache.set(store, verdict);
        if (verdict === 'mismatch') {
          console.error(`[mnemon] prime BLOCKING: schema-mismatch on store=${store}`);
          emitUnhealthyEvent(store, 'schema-mismatch');
        }
      }

      const phaseNote = phase === 'shadow'
        ? '\n\n[mnemon] Phase 1 shadow — recall not yet active for this group.'
        : '';
      return { hookSpecificOutput: { hookEventName: 'SessionStart', additionalContext: guide + phaseNote } };
    } catch (err) {
      const cls = classifyError(err as Error);
      if (cls.class === 'blocking') {
        console.error(`[mnemon] prime BLOCKING: ${cls.reason}`);
        emitUnhealthyEvent(store, cls.reason);
      } else {
        console.warn(`[mnemon] prime recoverable: ${cls.reason}`);
      }
      return {}; // safe default
    } finally {
      emitTurnMetric({ hook: 'prime', store, latencyMs: Date.now() - start });
    }
  };
}

export function createMnemonRemindHook(store: string) {
  return async (input: unknown) => {
    const start = Date.now();
    try {
      // Read schema verdict from cache populated at SessionStart (SF11). No re-detection.
      const schemaVerdict = schemaCache.get(store) ?? 'unknown';
      if (schemaVerdict === 'mismatch') {
        // Stay quiet on the hot path — the unhealthy event was already emitted at SessionStart.
        // Returning empty context lets the turn proceed normally.
        return {};
      }
      const phase = readPhase(store);
      const reminder = phase === 'shadow'
        ? '[mnemon] Phase 1 shadow — only consider remember; recall not yet active. Skip recall this turn.'
        : '[mnemon] Evaluate: recall needed? After responding, evaluate: remember needed?';
      return { hookSpecificOutput: { hookEventName: 'UserPromptSubmit', additionalContext: reminder } };
    } catch (err) {
      const cls = classifyError(err as Error);
      if (cls.class === 'blocking') {
        console.error(`[mnemon] remind BLOCKING: ${cls.reason}`);
        emitUnhealthyEvent(store, cls.reason);
      } else {
        console.warn(`[mnemon] remind recoverable: ${cls.reason}`);
      }
      return {};
    } finally {
      emitTurnMetric({ hook: 'remind', store, latencyMs: Date.now() - start });
    }
  };
}
```

**Approach:** `mnemon-backup.sh` runs nightly via host cron. **Code pattern:**

```bash
#!/bin/bash
# scripts/mnemon-backup.sh — nightly host cron
set -euo pipefail
DATE=$(date -u +%Y-%m-%d)
SOURCE_ROOT="${HOME}/.mnemon/data"
BACKUP_ROOT="${HOME}/backups/.mnemon-${DATE}"

mkdir -p "$BACKUP_ROOT"
for STORE_DIR in "$SOURCE_ROOT"/*/; do
  STORE=$(basename "$STORE_DIR")
  DB="${STORE_DIR}/mnemon.db"
  [ -f "$DB" ] || continue
  sqlite3 "$DB" ".timeout 5000" ".backup '${BACKUP_ROOT}/${STORE}.db'"
done

# Retention: keep last 7 daily + 4 weekly. Simple find with -mtime + a manifest of weekly anchors.
# Daily: delete daily snapshots older than 7 days unless they fall on a Sunday (weekly anchor).
# Weekly: keep Sunday snapshots up to 4 weeks back, delete older.
find "${HOME}/backups" -maxdepth 1 -type d -name '.mnemon-*' -mtime +7 \
  | while read OLD; do
      OLD_DATE=$(basename "$OLD" | sed 's/^.mnemon-//')
      OLD_DOW=$(date -u -d "$OLD_DATE" +%w)  # 0=Sunday
      OLD_AGE_DAYS=$(( ( $(date -u +%s) - $(date -u -d "$OLD_DATE" +%s) ) / 86400 ))
      # If it's a Sunday and within 4 weeks, keep it.
      if [ "$OLD_DOW" = "0" ] && [ "$OLD_AGE_DAYS" -lt 28 ]; then continue; fi
      rm -rf "$OLD"
    done

# scripts/mnemon-restore.sh
# Usage: mnemon-restore.sh <YYYY-MM-DD> <store>
set -euo pipefail
DATE=$1
STORE=$2
BACKUP_PATH="${HOME}/backups/.mnemon-${DATE}/${STORE}.db"
LIVE_PATH="${HOME}/.mnemon/data/${STORE}/mnemon.db"
[ -f "$BACKUP_PATH" ] || { echo "no backup at $BACKUP_PATH"; exit 1; }

# Optional: stop the running container or pause the agent group's writes — operator decides.
mkdir -p "$(dirname "$LIVE_PATH")"
mv "$LIVE_PATH" "${LIVE_PATH}.pre-restore-$(date +%s)" 2>/dev/null || true
cp "$BACKUP_PATH" "$LIVE_PATH"
echo "Restored ${STORE} from ${DATE} backup. Live store: ${LIVE_PATH}. Old store moved to ${LIVE_PATH}.pre-restore-*"
```

```typescript
import fs from 'fs';
import path from 'path';
import { execFile } from 'child_process';
import { promisify } from 'util';
import { GROUPS_DIR, DATA_DIR } from '../src/config.js';
import { readContainerConfig } from '../src/container-config.js';
const execFileP = promisify(execFile);

const STORES_DIR = path.join(DATA_DIR, 'mnemon-metrics', 'stores');
const TURNS_DIR = path.join(DATA_DIR, 'mnemon-metrics', 'turns');
const HEALTH_FILE = path.join(DATA_DIR, 'mnemon-health.json');

async function main() {
  fs.mkdirSync(STORES_DIR, { recursive: true });
  // Iterate enabled groups
  for (const folder of fs.readdirSync(GROUPS_DIR)) {
    const cfg = readContainerConfig(folder);
    if (!cfg.mnemon?.enabled || !cfg.agentGroupId) continue;
    const store = cfg.agentGroupId;
    try {

      const row = JSON.stringify({ ts: new Date().toISOString(), store, ...status }) + '\n';

    }
  }
  // Aggregate health from recent turn metrics
  const health: Record<string, { phase: 'ok' | 'unhealthy'; recent_unhealthy_events: Array<{ ts: string; reason: string }> }> = {};
  if (fs.existsSync(TURNS_DIR)) {
    for (const f of fs.readdirSync(TURNS_DIR)) {
      if (!f.endsWith('.jsonl')) continue;

          if (row.event_type !== 'unhealthy') continue;
          health[row.store] ??= { phase: 'ok', recent_unhealthy_events: [] };
          health[row.store].recent_unhealthy_events.push({ ts: row.ts, reason: row.reason });
        } catch { /* skip malformed */ }
      }
    }
  }
  // If a store has any unhealthy events in the last hour: mark unhealthy.
  const cutoff = Date.now() - 60 * 60 * 1000;
  for (const [store, h] of Object.entries(health)) {
    const recent = h.recent_unhealthy_events.filter(e => Date.parse(e.ts) > cutoff);
    h.recent_unhealthy_events = recent;
    h.phase = recent.length > 0 ? 'unhealthy' : 'ok';
  }
  fs.writeFileSync(HEALTH_FILE, JSON.stringify(health, null, 2) + '\n');
}

main().catch(e => { console.error(e); process.exit(1); });
```
