### Task A6: Create mnemon-impl.ts

**File:** `src/modules/memory/mnemon-impl.ts`
**Operation:** CREATE
**Test file:** `src/modules/memory/mnemon-impl.test.ts`

**Approach:** Implements `MemoryStore` by shelling out to mnemon CLI at `~/.local/bin/mnemon`. Uses `child_process.spawn` with `AbortController` for timeout-driven SIGTERM/SIGKILL lifecycle (M11 cycle-1). Calls A7's `redactSecrets` before every `remember` write. Per design §Detailed Design 1.

**Interface / signature:**
```typescript
export class MnemonStore implements MemoryStore {
  async recall(agentGroupId, query, opts = {}): Promise<RecallResult> { /* spawn + JSON parse + AbortController timeout */ }
  async remember(agentGroupId, fact, opts = {}): Promise<RememberResult> { /* idempotency key check + redact + spawn */ }
  async health(agentGroupId): Promise<{ ok, reason? }> { /* `mnemon status --store <id>` */ }
}
```

**ASSERT:** `recall()` spawns mnemon via `child_process.spawn`, never via `exec` (avoid shell injection).
**ASSERT:** `recall()` AbortController fires SIGTERM at `opts.timeoutMs` (default 1500), then SIGKILL after 500ms grace.
**ASSERT:** `recall()` returns `{ facts: [], totalAvailable: 0, ... }` on timeout / spawn error / parse error — never throws.
**ASSERT:** `remember()` short-circuits if `opts.idempotencyKey` already in process-local LRU cache (M3 cycle-3).
**ASSERT:** `remember()` calls `redactSecrets(fact)` before any mnemon CLI invocation; if redaction blocks, returns `{ action: 'skipped', factId: '' }` and increments health JSON's `redactionCount`.
**ASSERT:** No code path constructs a shell command string; all args passed as `spawn` argv array (cycle-1 retro #2 — defense against quoting bugs).

**Test cases:**
