### Severity breakdown

- **CRITICAL (1):** Bash secret sanitization hook (`createSanitizeBashHook`) — complete removal of ANTHROPIC_API_KEY / CLAUDE_CODE_OAUTH_TOKEN protection from every Bash invocation.
- **HIGH (6):** block-snowflake-connector hook, block-git-clone hook, email gate hook, `SDK_ENV_DENYLIST` (GRANOLA / EXA / BRAINTRUST secrets now leak into Bash env), model/effort switch machinery, system-prompt assembly (identity / formatting / capability manifest / file-delivery notes).
- **MEDIUM (12):** PreCompact thread archive + summary, self-approval-block, API-key fallback rotation, pre-flight session size compact, tool allowlist regression, `/workspace/thread` additional-directory, `buildMcpServers` (gitnexus/exa/granola/braintrust/omni/ollama built-in registrations), plugin discovery, `send_file` allowlist+dedup+sizecap, `render_diagram`, scheduling feature gaps, group admin tools, `read_thread`/`read_thread_by_key`, memory tools, ollama MCP bridge, terminal error classification (`prompt_too_long`), IPC env exposure for plugin hooks.
- **LOW (8):** Checkpoint nudge, stall detector, slash command handling, buildPromptContent formatting, pipe_ack, project tracking tools (ship-log / backlog / activity / scan_commits / update_plugin), tone profiles, git worktree tools, workflow-plugin safety notice.
- **N/A (3):** IPC_INPUT_DIR / `_close` sentinel (architectural replacement by DB polling), `/tmp/input.json` cleanup, `disallowedTools` change (intentional improvement).

### Highest-leverage ports, in order

1. `createSanitizeBashHook` — single easiest CRITICAL win. Re-register a `PreToolUse` Bash hook in `providers/claude.ts` that prepends `unset` for the relevant secret names.
2. `SDK_ENV_DENYLIST` — 3-line change; denylist GRANOLA_ACCESS_TOKEN / EXA_API_KEY / BRAINTRUST_API_KEY when building the env passed to `sdkQuery`.
3. Plugin discovery — if `nanoclaw-plugin/` is currently dead code (no plugins: passed to the SDK), every other PreToolUse/PostToolUse hook in v2 is silently inactive. This is both a fix and a diagnostic for what else is broken.
4. System-prompt assembly — the identity note and capability manifest matter materially for agent behavior; the dropped "never produce a meta-response" note causes the well-known "No response requested." failure mode.
5. `createBlockSnowflakeConnectorHook` and `createBlockGitCloneHook` — both cheap to re-port once (1) is in place and both close real bypass paths.
